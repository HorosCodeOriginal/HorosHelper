# Codebases erkunden

Hybrid-Suchtool: Geschwindigkeit von `ripgrep` plus strukturelles Bewusstsein von `tree-sitter` oder `_MAP.md`. Findet Treffer und liefert die *gesamte* Funktion oder Klasse mit dem Match, semantisch dedupliziert.

## Progressive Disclosure

**Standard: nur Signaturen** (Docstrings + Deklarationen ohne Funktionskörper), 10–20× weniger Tokens. `--expand-full` für vollständige Implementierungen.

## Installation

```bash
uv venv /home/claude/.venv
# tree-sitter is optional if using --use-maps mode
uv pip install tree-sitter-language-pack --python /home/claude/.venv/bin/python
```

## Nutzung

```bash
# Default: signatures only with tree-sitter (efficient)
/home/claude/.venv/bin/python /mnt/skills/user/exploring-codebases/scripts/search.py "query" /path/to/repo

# Full implementations
/home/claude/.venv/bin/python /mnt/skills/user/exploring-codebases/scripts/search.py "query" /path/to/repo --expand-full

# Map-based mode: no tree-sitter required (#276)
# Requires _MAP.md files generated by mapping-codebases
/home/claude/.venv/bin/python /mnt/skills/user/exploring-codebases/scripts/search.py "query" /path/to/repo --use-maps
```

## Optionen

- `--glob pattern`: Dateien filtern (z. B. `*.py`, `*.ts`)
- `--expand-full`: Vollständige Implementierungen statt Signaturen zurückgeben
- `--json`: JSON-Ausgabe für maschinelle Verarbeitung (Standard ist Markdown)
- `--use-maps`: Vorgenerierte `_MAP.md`-Dateien statt tree-sitter für Kontexterweiterung nutzen. Vermeidet redundantes tree-sitter-Parsing, wenn Maps bereits existieren.

## Map-basierter Modus (v0.3.0)

Wenn `_MAP.md`-Dateien existieren (generiert von `mapping-codebases`), nutze `--use-maps`, um tree-sitter komplett zu überspringen:

1. `mapping-codebases` führt tree-sitter einmal aus, um `_MAP.md`-Dateien mit Symbolpositionen zu erzeugen
2. `exploring-codebases --use-maps` nutzt ripgrep + Map-Daten für Kontexterweiterung
3. Kein redundantes AST-Parsing zur Suchzeit

**Vorteile:**
- Einmalige tree-sitter-Ausführung pro Codebase (bei Map-Generierung)
- Schnellere Suchen (kein AST-Parsing-Overhead)
- `tree-sitter-language-pack` zur Laufzeit nicht erforderlich
- Maps dienen als kanonischer Index für Navigation und Suche

**Empfohlener Workflow:**
```bash
# Step 1: Generate maps (one-time, or after code changes)
python /mnt/skills/user/mapping-codebases/scripts/codemap.py /path/to/repo

# Step 2: Search using maps (fast, no tree-sitter needed)
python /mnt/skills/user/exploring-codebases/scripts/search.py "query" /path/to/repo --use-maps
```

## Umfang und Grenzen

**Liefert strukturelle Code-Elemente** — Funktionen, Klassen, Methoden, Interfaces, Enums, Structs, Traits und Module in 11 unterstützten Sprachen (Python, JavaScript, TypeScript, Go, Rust, Ruby, Java, C, C++, PHP, C#).

**Liefert nicht:**
- Import-/Require-Statements
- Modulweite Variablenzuweisungen oder Konstanten
- Standalone-Decorators (Decorators an Funktionen/Klassen sind mit dem Parent enthalten)
- Type-Aliase oder standalone Type-Annotations
- Kommentare außerhalb von Funktionen/Klassen

Für diese nicht-strukturellen Elemente nutze direkt plain `ripgrep` (via Grep-Tool).
