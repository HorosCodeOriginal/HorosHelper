# Dokumentations-Styleguide

## Überschriften
- Satzschreibung verwenden
- Überschriften kurz und präzise halten

## Code-Blöcke
- Immer eine Sprache angeben
- Beispiele minimal und ausführbar halten

## Links
- Relative Links für interne Docs verwenden
- Stabile URLs für externe Referenzen bevorzugen
