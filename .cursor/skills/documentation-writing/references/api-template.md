# API-Dokumentationsvorlage

## Endpoint: /resource

**Method:** GET | POST | PUT | DELETE

**Description:** Was der Endpoint macht.

### Parameters
- `id` (string, required) - Ressourcen-Identifier

### Response
```json
{
  "data": {}
}
```

### Errors
- `400` Bad Request
- `401` Unauthorized
- `404` Not Found
